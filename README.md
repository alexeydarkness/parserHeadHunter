# HH Parser

Парсер вакансий с [hh.ru](https://hh.ru) на Java. Делает поисковый запрос, собирает карточки вакансий со страниц выдачи и выгружает результат в Excel-файл.

## Возможности

- Поиск вакансий по любому запросу (вводится с клавиатуры)
- Парсинг названия, компании, зарплаты и опыта из карточек выдачи
- Обход нескольких страниц с задержкой между запросами
- Рандомный User-Agent из пула, чтобы снизить шанс блокировки
- Выгрузка результата в `.xlsx`

## Стек

- **Java 17**
- **[Jsoup](https://jsoup.org/)** — HTTP-запросы и парсинг HTML
- **[Apache POI](https://poi.apache.org/)** — запись в Excel
- **Maven** — сборка и зависимости

## Установка

Нужны установленные **JDK 17+** и **Maven** (или IntelliJ IDEA со встроенным Maven).

```bash
git clone https://github.com/USERNAME/hh-parser.git
cd hh-parser
mvn clean compile
```

Зависимости (Jsoup и Apache POI) подтянутся автоматически из `pom.xml`.

## Запуск

```bash
mvn exec:java -Dexec.mainClass="com.todo.HhParser"
```

Или запусти метод `main` в классе `HhParser` прямо из IDE.

После запуска программа спросит:

```
Какую вакансию ищем? java разработчик
Сколько страниц спарсить? 2
```

По завершении в корне проекта появится файл `vacancies.xlsx` с таблицей вакансий.

## Как это работает

1. Запрос и число страниц читаются с клавиатуры, запрос URL-кодируется под кириллицу и пробелы.
2. Для каждой страницы отправляется GET-запрос через Jsoup с рандомным User-Agent и браузерными заголовками.
3. Карточки вакансий ищутся по стабильным атрибутам `data-qa`, а не по динамическим CSS-классам — их hh.ru часто меняет.
4. Из каждой карточки достаются поля, собираются в список объектов `Vacancy`.
5. Список записывается в `.xlsx` через Apache POI.

## Ограничения

- hh.ru может менять разметку и атрибуты `data-qa` — тогда селекторы в `findVacancyCards` и `parseCard` придётся обновить.
- Сайт защищён от массового парсинга: при частых запросах возможна временная блокировка. Задержка между страницами это смягчает, но не гарантирует.
- Для стабильного и «легального» доступа к данным лучше использовать [официальный API hh.ru](https://api.hh.ru/), отдающий JSON.
